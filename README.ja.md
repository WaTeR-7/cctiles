# CC-Tiles

[![License: MIT OR Apache-2.0](https://img.shields.io/badge/license-MIT%20OR%20Apache--2.0-blue.svg)](#license)
[![CI](https://github.com/WaTeR-7/cctiles/actions/workflows/ci.yml/badge.svg)](https://github.com/WaTeR-7/cctiles/actions/workflows/ci.yml)

[Claude Code](https://github.com/anthropics/claude-code) の CUI セッションを並列で実行・監視するための TUI（テキストユーザーインターフェース）アプリです。Rust で実装しています。

`cctiles` を起動し、画面分割数と各ペインのディレクトリを選ぶと、各ペイン（**タイル**）がそれぞれ自分の Claude Code セッションを起動・実行します。各タイルにはそのセッションが今何をしているかが一目で分かるライブサマリーが表示されるので、大量のターミナルタブを行き来しなくても複数のエージェントを並行して走らせておけます。

![cctilesのデモ: 2x2グリッドの各タイルにライブの状態インジケータ・git情報・スクロールするアクティビティフィードが表示され、その後フローティングターミナルとキーバインドのヘルプ画面を開く様子](docs/demo.gif)

## 機能

- **複数セッションのグリッド表示** - 任意の行数・列数でレイアウトでき、各タイルが独立した `claude` プロセスを実行します。
- **一目で分かる状態表示** - 各タイルのタイトルにある色付きインジケータで、そのセッションが今「アイドル」「作業中」「許可待ち/質問待ち」「バックグラウンドタスク実行中」「クラッシュ」のどれかが分かります。画面をスクレイピングするのではなく Claude Code 自体の [hooks](https://code.claude.com/docs/en/hooks.md) 機能で駆動しているため、画面を見ただけでは分からないバックグラウンドタスクの状態も正確に反映されます。
- **ライブアクティビティフィード** - 各タイルに直近のツール呼び出しやメッセージの要約が流れるように表示され、タイルを開かなくてもセッションが何をしているか把握できます。
- **タイルごとのgit状態** - ディレクトリがgitリポジトリであれば、現在のブランチと作業ツリーの差分（例: `main  +12/-3`）を表示します。
- **フローティングターミナル** - タイルで `Enter` を押すと、そのセッションを操作できる本物のインタラクティブなターミナルがグリッドの上にフローティングで開きます（グリッドの他の部分は見えたままです）。`Ctrl+G` でグリッドに戻れます。
- **クラッシュからの復帰** - セッションが予期せず終了した場合、タイルが単に空白になるのではなく明確なエラー表示になり、その場で再起動できます。

## 必要環境

- [Claude Code CLI](https://github.com/anthropics/claude-code)（`claude` コマンド）がインストールされ、`PATH` から実行できること。
- ターミナルエミュレータ（cctiles はターミナル上で直接動かす TUI アプリです）。

## インストール

まだ crates.io には公開していません。現時点ではソースからビルドしてください。

```sh
git clone https://github.com/WaTeR-7/cctiles.git
cd cctiles
cargo build --release
./target/release/cctiles
```

## 使い方

初回起動時（または `--setup` オプション付きで起動したとき）は、簡単なセットアップ画面が表示されます。

1. グリッドサイズ（行数・列数）を選ぶ
2. 各タイルに対応するディレクトリを選ぶ（それぞれ独立した `claude` セッションになります）

選んだ内容は設定ファイル（[設定](#設定)を参照）に保存され、次回起動時にそのまま使われます。

### キーバインド

**グリッド画面**

| キー                          | 動作                               |
| ----------------------------- | ---------------------------------- |
| `h` `j` `k` `l` / 矢印キー     | タイル間でフォーカスを移動         |
| `Enter`                        | フォーカス中のタイルをフローティングターミナルで開く |
| `r`                            | フォーカス中のタイルのセッションを再起動 |
| `x`                            | フォーカス中のタイルのセッションを終了   |
| `?`                            | キーバインドのヘルプ表示を切り替え |
| `q`                            | 終了                               |

**フローティングターミナル**

| キー       | 動作                                             |
| ---------- | ------------------------------------------------ |
| （入力）   | そのままセッションに転送されます（直接入力しているのと同じ） |
| `Ctrl+G`   | グリッドに戻る（セッションは動いたままです）       |

## 設定

設定はTOML形式で、デフォルトではお使いのOSの標準的な設定ディレクトリ（Linuxの例: `~/.config/cctiles/config.toml`）に保存されます。

```toml
rows = 2
cols = 2
tile_dirs = [
    "/path/to/project-a",
    "/path/to/project-b",
    "/path/to/project-c",
    "/path/to/project-d",
]
```

`--config <path>` で別のファイルを指定したり、`--setup` で既存の設定を残したまま対話式セットアップをやり直したりできます。

## 既知の制限

- 同じディレクトリを同時に複数のタイルで指定すると、状態表示とアクティビティフィードが混線することがあります（[#68](https://github.com/WaTeR-7/cctiles/issues/68)）。現時点では各タイルに別々のディレクトリを指定してください。

今後の予定は[未解決のIssue一覧](https://github.com/WaTeR-7/cctiles/issues)を参照してください。
